# 🏗️ دليل بناء شات فلو متكامل مع CRM — خطوة بخطوة (الإصدار الجديد)

> الدليل ده هيمشيك من الصفر لحد ما تبني شات فلو شغال بالكامل — **Tool Agent واحد** مع أدوات CRM مباشرة (إنشاء أوامر، مراقبة محادثات، بحث في المنتجات). العمارة الجديدة أسرع وأخف وأسهل في الصيانة.

---

## ⚡ ليه العمارة الجديدة؟

| القديم                                                 | الجديد                                               |
| ------------------------------------------------------------ | ---------------------------------------------------------- |
| 3 Agents (Tool + AppCity + ERP)                              | **Tool Agent واحد**                              |
| 7-9 نودز في الشات فلو                          | **6 نودز** (أو 4 لو مش بتستخدم RAG) |
| 3 برومبتات منفصلة (Sales + ERP + Orchestrator) | **برومبت واحد** فيه كل حاجة       |
| الاستجابة: 5-8 ثواني                           | الاستجابة:**1-3 ثواني**                |
| الخطأ بيضيع بين الطبقات                  | الخطأ مباشر وواضح                           |

## ⚙️ ثالثاً: إعدادات النودز

### 3.1 — Chat Model

- اختار الموديل المناسب (Gemini 2.5 Flash أو GPT-4.1 Mini)
- Temperature: `0.3` (قليل عشان الـ Agent يلتزم بالتعليمات)
- Max Tokens: `2000` (أو أعلى لو البرومبت طويل)

### 3.2 — Buffer Memory

- Window Size: `20`
- Session ID: `$sessionId`

### 3.3 — CRM Order Tool

| الخانة             | القيمة                                                    |
| ------------------------ | --------------------------------------------------------------- |
| **CRM Base URL**   | `https://crm.octobot.it.com`                                  |
| **API Key**        | انسخه من CRM Dashboard ← Integration Keys ← Orders tab |
| **Variables**      | اختار`Session ID`                                        |
| **Orders Actions** | اختار`Create Order`                                      |
| **Tool Name**      | `create_crm_order`                                            |

### 3.4 — CRM Monitoring Tool (v3.0)

| الخانة           | القيمة                     |
| ---------------------- | -------------------------------- |
| **CRM Base URL** | `https://crm.octobot.it.com`   |
| **API Key**      | نفس مفتاح CRM Order Tool |
| **Variables**    | اختار`Session ID`         |

> ⚠️ الإصدار 3.0 مش محتاج Chat Model ولا Monitoring Prompt Template منفصل. الـ Agent هو اللي بيحلل المحادثة والـ Tool بيبعت النتيجة للـ CRM مباشرة.

### 3.5 — WhatsDeveloper (اختياري)

- اختار Credential: `whatsDeveloperApi`
- Type: `Messaging`
- Messaging Actions: `Send Text Message`

## 📝 رابعاً: بناء البرومبت — إزاي تدمج تعليمات CRM مع برومبت البيع

### 4.1 — الفكرة الأساسية

برومبت واحد في نود **Chat Prompt Template** بيحتوي على **كل حاجة**:

```
┌─────────────────────────────────────────┐
│         Chat Prompt Template            │
│                                         │
│  §0 — هوية البوت وقواعده الأساسية       │
│  §1 — الأدوات (CRM Tools + Pricing)     │  ← جزء CRM
│  §2 — الترحيب                           │
│  §3 — البيع (Sales Playbook)            │
│  §4 — المعرفة (Knowledge Base)          │
│  §5 — تدفق الأوردر مع CRM               │  ← جزء CRM
│  §6 — اللهجة المصرية                     │
│  §7 — المراقبة (Monitoring Templates)    │  ← جزء CRM
│  §8 — استخراج البيانات (Extraction Keys) │  ← جزء CRM
│  §9 — قواعد التنبيه (Alert Rules)        │  ← جزء CRM
│  §10 — الشكاوى والاستفسارات              │
└─────────────────────────────────────────┘
```

### 4.2 — إزاي تبني البرومبت

#### لو بتعمل شات فلو جديد من الصفر:

انسخ محتوى ملف **`WEARFIT_CRM_PROMPT_V2.txt`** كامل وحطه في نود **Chat Prompt Template** ← خانة **System Message**.

ده برومبت جاهز فيه:

- البيع (منتجات، تسعير، مقاسات، ألوان، ترحيب، معالجة الاعتراضات، لهجة مصرية)
- CRM (رفع أوامر، مراقبة، استخراج بيانات، تنبيهات)

#### لو عندك برومبت Sales شغال وعايز تضيف عليه CRM بس:

1. خد برومبتك الحالي (زي `WEARFIT_CRM_PROMPT.txt` = برومبت مبيعات بس من غير CRM)
2. افتح ملف **`WEARFIT_CRM_PROMPT_V2.txt`** ← ده برومبت مبيعات + CRM كامل
3. شوف الأجزاء دي بالظبط وانسخها في برومبتك:

| القسم                                | اسمه في V2     | اللي بيقدمه                                                             |
| ----------------------------------------- | -------------------- | --------------------------------------------------------------------------------- |
| **أدوات CRM**                  | §1 — أول جزء | تعريف create_crm_order و crm_monitoring_note                                |
| **تدفق الأوردر**         | §5 — Phase 3       | الـ 4 خطوات: CRM → Monitoring → WhatsApp → Confirm                     |
| **قوالب المراقبة**     | §7                  | 6 سيناريوهات جاهزة لـ monitoring كل رسالة                 |
| **مفاتيح الاستخراج** | §8                  | جدول الـ 16 مفتاح اللي بيتستخرجوا من المحادثة |
| **قواعد التنبيه**       | §9                  | 5 alert rules مع أمثلة للتطبيق                                      |

> 💡 **نصيحة:** استخدم ملفين منفصلين لو عايز تنظيم أفضل:
>
> - `BRAND_SALES.txt` ← أجزاء البيع بس (§0, §2, §3, §4, §6)
> - `BRAND_CRM.txt` ← أجزاء CRM بس (§1 Tools, §5 Phase 3, §7, §8, §9)
>
> ولما تخلص تعديل، ادمجهم في ملف واحد (`BRAND_FULL.txt`) وحطه في Chat Prompt Template.

## 🗝️ خامساً: المفاتيح المستخلصة (Extracted Keys) — إزاي بتشتغل

### 5.1 — آلية العمل تلقائياً

**مش محتاج تعرف المفاتيح في أي مكان تاني غير البرومبت.** الآلية كالآتي:

```
1. الـ Agent (حسب تعليمات §7 و §8 في البرومبت) بيحلل رسالة العميل
   ↓
2. الـ Agent بينادي crm_monitoring_note ومعه keys = {{"client_name": "محمد", "phone": "01xxx", ...}}
   ↓
3. الـ CRMMonitoringTool بيستقبل الـ keys
   ↓
4. الـ Tool تلقائياً بيبني keyDefinitions من أسماء المفاتيح اللي الـ Agent باعتها
   ↓
5. الـ Tool بيبعت keyDefinitions + keys للـ CRM backend
   ↓
6. الـ CRM backend بيسجل المفاتيح الجديدة في monitoring_key_definitions
   وبعدين بيخزن القيم في session_extracted_values
   ↓
✅ البيانات ظهرت في بروفايل العميل على CRM Dashboard
```

### 5.2 — إزاي تتحكم في المفاتيح

**كل التحكم من البرومبت.** المفاتيح اللي تكتبها في جدول §8 هي اللي الـ Agent هيستخدمها.

#### جدول المفاتيح للعميل الحالي (موجود في §8 من البرومبت):

| Key               | Type | Extract When                                          |
| ----------------- | ---- | ----------------------------------------------------- |
| `interest`      | text | العميل بيسأل عن منتج/سعر/مقاس |
| `product_qty`   | text | العميل ذكر عدد القطع                 |
| `product_color` | text | العميل اختار/ذكر لون                 |
| `product_size`  | text | العميل قال مقاسه أو وزنه          |
| `client_name`   | text | العميل قال اسمه                          |
| `phone`         | text | العميل قال رقم تليفونه             |
| `governorate`   | text | العميل قال المحافظة                  |
| `address`       | text | العميل قال العنوان                    |
| `base_price`    | text | تم حساب سعر المنتجات                 |
| `shipping_fee`  | text | تم حساب الشحن                              |
| `order_value`   | text | الإجمالي مع الشحن                      |
| `objection`     | text | العميل عنده اعتراض                    |
| `stall_reason`  | text | العميل بيأجل/بيقول هيفكر         |
| `competitor`    | text | العميل ذكر شركة منافسة             |
| `anger_reason`  | text | العميل غاضب                                 |
| `order_id`      | text | رقم الطلب من CRM                            |

> 📝 **لإضافة مفتاح جديد:** ضيف صف في الجدول في §8، وضيف المفتاح ده في قوالب §7 في المكان المناسب. الـ system هيسجله تلقائياً أول ما الـ Agent يبعت فيه قيمة.

## 🚨 سادساً: قواعد التنبيه (Alert Rules) — إزاي بتشتغل

### 6.1 — آلية العمل

كل رسالة من العميل، الـ Agent بيقيّم 5 قواعد (حسب §9 في البرومبت). لو قاعدة اتحققت، بيبعت alert_level و alert_reason مع crm_monitoring_note.

### 6.2 — قواعد التنبيه الحالية

| # | Rule Name                               | Alert Level       | Condition                                                                                               |
| - | --------------------------------------- | ----------------- | ------------------------------------------------------------------------------------------------------- |
| 1 | عميل غاضب                       | **danger**  | العميل يعبر عن غضب شديد أو إحباط أو يهدد بعدم التعامل        |
| 2 | طلب تسجيل                       | **warning** | العميل يوافق ويعطي اسمه ورقمه — انتقل لمرحلة إتمام الطلب |
| 3 | ذكر منافس                       | **warning** | العميل يذكر اسم شركة منافسة ويقارن الأسعار                          |
| 4 | مهتم بالتسعير               | **warning** | العميل يسأل عن الأسعار أو العروض أو الخصومات                       |
| 5 | عميل متردد ويوشك يخرج | **danger**  | العميل يعبر عن تردد شديد أو يقول أنه سيفكر ويعود لاحقاً    |

> 📝 **لإضافة قاعدة جديدة:** ضيف صف في جدول §9 في البرومبت. الـ Agent هيبدأ يطبقه تلقائياً من غير أي تعديل في النودز.

## 🔑 سابعاً: تجهيز API Key من CRM Dashboard

### 7.1 — افتح CRM Dashboard

```
https://crm.octobot.it.com/dashboard/integration
```

### 7.2 — تبويب الطلبات (Orders)

1. اعمل **Create Key** للعميل
2. انسخ الـ API Key (بيبدأ بـ `crm_`)
3. حطه في **CRM Order Tool** ← خانة API Key
4. حطه في **CRM Monitoring Tool** ← خانة API Key

### 7.3 — الأعمدة (Key Mapping)

1. دوس على **عرض خريطة المفاتيح** ← **تعديل**
2. ضيف الأعمدة اللي عايزها — كل عمود = حقل في شيت الأوامر
3. **الأعمدة دي هي هي المفاتيح اللي هتستخدمها في create_crm_order في البرومبت**

#### مثال للأعمدة:

| Key Name           | Display Name            | Type     |
| ------------------ | ----------------------- | -------- |
| `clientName`     | اسم العميل     | text     |
| `clientPhone`    | رقم الهاتف     | text     |
| `clientBckPhone` | رقم إضافي       | text     |
| `governorate`    | المحافظة        | text     |
| `clientAddress`  | العنوان كامل | text     |
| `productQty`     | الكمية            | number   |
| `productColor`   | اللون              | text     |
| `productSize`    | المقاس            | text     |
| `orderDetails`   | تفاصيل الطلب | text     |
| `basePrice`      | سعر المنتجات | number   |
| `status`         | حالة العميل   | dropdown |
