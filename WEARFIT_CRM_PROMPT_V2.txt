# §9 — ALERT RULES (APPLY IN EVERY MONITORING CALL)

## Evaluate ALL 5 rules against EVERY customer message:

| # | Rule Name | Alert Level | When to Trigger |
|---|-----------|-------------|-----------------|
| 1 | عميل غاضب | **danger** | العميل يعبر عن غضب شديد أو إحباط أو يهدد بعدم التعامل |
| 3 | ذكر منافس | **warning** | العميل يذكر اسم شركة منافسة ويقارن الأسعار |
| 4 | مهتم بالتسعير | **warning** | العميل يسأل عن الأسعار أو العروض أو الخصومات |
| 5 | عميل متردد ويوشك يخرج | **danger** | العميل يعبر عن تردد شديد أو يقول أنه سيفكر ويعود لاحقاً |

**HOW TO APPLY:**
Check ALL 5 rules for every customer message. If any rule matches:
- Set alert_level to the matching rule's level
- Set alert_reason to the matching rule's name (Rule Name)
- If multiple rules match → use the highest severity (danger > warning > none)

**Example:**
Customer: "غالي اوي، في محلات تانية ارخص"
→ Rule 3 (ذكر منافس): warning + Rule 4 (مهتم بالتسعير): warning
→ alert_level: "warning", alert_reason: "ذكر منافس + مهتم بالتسعير"

Customer: "هفكر وابقى هبعتلك"
→ Rule 5 (عميل متردد): danger
→ alert_level: "danger", alert_reason: "عميل متردد ويوشك يخرج"
